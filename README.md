## 项目背景

在智能温室或植物生长监测场景中，使用普通单目摄像头进行叶面积估算时，相机姿态（高度、角度）的微小变化会导致显著的透视失真，从而使像素面积与实际物理面积之间产生较大偏差。

本项目旨在探索一种低成本方案：利用打印的棋盘格标定板和单目相机，通过姿态估计和尺度补偿，尝试缓解透视失真对叶面积测量的影响，实现像素面积向参考姿态下等效面积的粗略还原。

该方法不依赖深度传感器，仅使用普通USB摄像头和9×6棋盘格，适用于姿态变化相对可控的场景（如实验室/温室盆栽固定拍摄等固定拍摄场景）。
在小范围姿态扰动下，它能有效降低误差并改善趋势监测的稳定性，但在大姿态变化或复杂叶片结构场景下，补偿效果有限。

## 方法概述

算法流程如下：

1. **植物区域检测与分割**  
   - 使用SAM3对植物叶片面积进行分割，输出像素面积

2. **相机姿态估计**  
   - 检测棋盘格角点（cv2.findChessboardCorners + cornerSubPix）  
   - 通过solvePnP算法计算相机外参（旋转向量rvec、平移向量tvec）  
   - 以名为“1”的图像作为参考姿态，计算当前图像相对于参考的姿态变化（高度差、水平距离差、俯仰角差）

3. **尺度补偿与面积还原**  
   - 计算三种补偿因子：  
     - 基于棋盘格投影像素间距的尺度因子  
     - 基于PnP姿态估计的尺度因子  
     - PnP尺度因子结合俯仰角cos补偿（cos(θ_ref) / cos(θ_current)）  
   - 还原面积 = 当前像素面积 × (补偿因子)²  
   - 输出误差对比表（相对于参考图像的最大叶面积）

## 实验结果与误差分析

在实际环境中采集的多组数据表明：

- 当相机姿态变化较小（高度变化≤15 cm，俯仰角变化≤5°）时：  
  - 未补偿误差约10-25%  
  - 采用PnP姿态补偿后，误差可降低至5–15%。

- 当姿态变化较大（高度变化≈15 cm，俯仰角变化≈15°）时：  
  - 未补偿误差可达130%以上  
  - PnP姿态补偿后误差仍为80–100%，角度补偿项在大角度下反而加剧偏差

主要误差来源包括：

- 棋盘格在图像中占比过小导致角点检测精度下降，进而放大PnP估计噪声  
- 俯仰角补偿的cos项在极端角度下可能失效
- 叶片非严格共面（翘起、重叠、高低差），单一尺度因子无法同时适用于所有区域  
- 分割模型边缘误差（约5–10%）  
- 镜头残余畸变及光照条件变化

## 项目结构说明

本项目的核心文件与文件夹功能如下：

### 核心代码
- **`sam3_area_calculation.py`**  
  **核心主程序**。使用 SAM3 模型进行高精度植物分割，结合 PnP (Perspective-n-Point) 算法计算相机位姿（距离、高度、角度），并据此对植物面积进行还原补偿。  
  *支持生成带有详细测量数据（距离、高度差、角度差、面积）和棋盘格可视化（角点连线、坐标轴）的标注图片。*

- **`calibration_utils.py`**  
  **标定工具库**。包含棋盘格角点检测、相机内参加载、PnP 位姿解算、像素比例因子计算等核心数学逻辑。是 `sam3_area_calculation.py` 的底层支持模块。

- **`sam3_qt_app.py`**  
  **可视化 GUI 工具**。基于 PyQt5 的图形界面程序，允许用户交互式地加载图片、输入提示词（Prompt）并查看 SAM3 的分割效果，适合快速测试模型性能。

- **`pnp.py`**  
  **旧版/对比算法**。基于 YOLOv8 + DeeplabV3+ 的面积还原实现。用于与 SAM3 版本进行效果对比。

- **`camera_calibration.py`**  
  **相机内参标定脚本**。用于根据采集的棋盘格图片计算相机的内参矩阵和畸变系数，生成 `camera_params.npz` 文件。

- **`Image_segmentation_prediction.py`**  
  DeeplabV3+ 模型的推理封装类，主要被 `pnp.py` 使用。

### 数据与资源

- **`output/`**  
  **结果输出目录**。运行脚本后生成的 CSV 数据报表（`*面积对比.csv`）和带有可视化标注的图片（`annotated_*.jpg`）都会保存在此。

- **`sample/`**  
  **测试数据集**。存放用于测试的植物图片文件夹，通常包含一张名为 `1.jpg` 的参考图片。

- **`calibration_images/`**  
  用于相机内参标定的原始棋盘格图片集。

- **`camera_params.npz`**  
  相机内参数据文件，由 `camera_calibration.py` 生成，是后续所有 PnP 计算的基础，可结合样本进行使用。
